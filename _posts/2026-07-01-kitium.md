## Overview

**kitium** is a CI/CD gate that converts Altium PCB projects to KiCad and validates them — DRC, gerbers, renders, BOM cross-check — on every pull request. Source: [Cimos/kitium](https://github.com/Cimos/kitium). MIT-licensed.

The name blends **Ki**(Cad) with (Al)**tium**, with a nod to Zeno of Citium, the founder of Stoicism — which is about right for a tool doing dispassionate, rule-based design review with no opinions about your layout.

## The idea

If your team designs in Altium but you want KiCad's free, scriptable, headless toolchain for review, you don't want to migrate — you want a robot that re-derives KiCad artefacts from the Altium source on every PR and tells you if anything's wrong. Altium stays the authoritative source; the generated KiCad files are never committed back.

## What it checks

On each run it:

1. Locates the Altium project (`.PrjPcb` / `.PcbDoc`), including multi-board setups.
2. Converts each board headlessly with `kicad-cli pcb import --format altium`.
3. Runs it through KiBot: DRC, gerber generation, 2D/3D renders, and visual comparisons.
4. Cross-checks the board BOM against an exported Altium CSV.
5. Posts the results back to the pull request — renders, diff tables, and downloadable artefacts.

## What's solid, partial, and not done

I'd rather be honest about maturity than oversell it:

- **Solid** — PCB conversion, DRC, gerbers, and renders. These are native, headless operations and they work.
- **Partial** — visual diff (config is in place; base-reference wiring is still pending) and BOM validation (it leans on the Altium CSV cross-check, because KiCad can't recover MPN/supplier metadata from a converted board).
- **Not implemented** — schematic ERC. It needs fragile GUI automation, so it's a non-blocking stretch goal, not a promise.

## Using it in CI

It's a Docker-based GitHub Action built on the `kicad/kicad:10` container with KiBot and the KiCad CLI, shipped as a pre-built GHCR image. Drop it in `.github/workflows/pcb-gate.yml`:

```yaml
- uses: Cimos/kitium@v0.1.0
  with:
    project: hardware/MyBoard.PrjPcb
    bom_csv: hardware/MyBoard_BOM.csv
    drc: report          # or "block"
    github_token: ${{ secrets.GITHUB_TOKEN }}
```

`drc: block` turns a failing check into a failing gate; `report` just comments.

## Status

Currently **v0.1.0** — end-to-end PCB validation is verified on real Altium fixtures. The internals are `entrypoint.sh` (orchestration), `convert.sh` (multi-board loop), `bom_crosscheck.py`, and `kitium.kibot.yaml` if you want to read how it hangs together.
